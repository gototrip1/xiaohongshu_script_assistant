# 小红书博主调研与脚本创作方法论说明

> 文档说明：本文档阐述小红书 Agent 系统在"找博主 → 参考内容 → 设计脚本与分镜"全流程中的工作方法与设计思路。

> 案例参考：[轻醒_小红书博主调研报告.md](/docs/light醒_小红书博主调研报告.md)

## 一、整体流程概览

系统采用**两段式 Agent 协作架构**，由主代理调度两个专业子代理完成全流程：

```
用户输入 brief
    │
    ▼
┌─────────────────────────────────┐
│  主代理（main_agent）           │
│  负责意图识别与任务分发         │
└──────────┬──────────────────────┘
           │
     ┌─────┴──────┐
     ▼            ▼
┌─────────┐  ┌──────────────┐
│investi- │  │script_       │
│gation   │→  │generate      │
│博主调研  │  │脚本创作       │
└─────────┘  └──────────────┘
   产出：         产出：
   10位博主       口播文案
   推荐1位        分镜设计
   内容拆解        飞书文档
   提示词         风险质检
```

| 阶段 | 子代理 | 核心工具 | 产出物 |
|------|--------|---------|--------|
| 第一阶段：博主调研 | investigation | `web_search` + 爬虫 skill | 10 位博主清单、推荐 1 位、内容拆解、创作提示词 |
| 第二阶段：脚本创作 | script_generate | 飞书 15 个 MCP 工具 + generate_content skill | 口播文案、产品植入点、分镜表、飞书文档 |

## 二、怎么找博主（investigation 子代理）

### 2.1 搜索策略：多维关键词矩阵

系统并非单一关键词搜索，而是按**品类、场景、痛点**三个维度构建搜索矩阵，进行多轮搜索：

| 搜索维度 | 关键词模板 | 示例（以轻食酸奶为例） |
|---------|-----------|---------------------|
| 品类关键词 | "小红书 {品类} 博主推荐" | "小红书 酸奶 博主推荐" |
| 场景关键词 | "{场景} 小红书 达人" | "早餐 小红书 达人" |
| 痛点关键词 | "{关键词} 种草 博主" | "减脂控糖 种草 博主" |

多轮搜索的目的：**避免单一搜索结果偏差**，确保覆盖不同类型的博主（如案例中同时覆盖了健身教练型、减脂打卡型、上班族生活型、轻食美学型四类博主）。

### 2.2 筛选标准：五维评估

对搜索到的博主，按以下 5 个维度进行筛选，最终保留 10 位：

1. **真实活跃**：近期有持续更新，非停更账号
2. **内容风格适合自然植入**：非纯硬广型博主，有日常分享内容
3. **粉丝画像与目标人群匹配**：年龄段、性别、消费力、兴趣标签
4. **互动数据健康**：点赞/收藏/评论比例合理
5. **与 brief 关联点明确**：能用一句话说明入选原因

### 2.3 信息记录：结构化档案

每位博主记录以下信息，形成结构化档案（见案例报告第二章）：

```
博主昵称 → 主页链接 → 内容方向 → 粉丝画像 → 近期2-3条代表内容（标题+互动数据）
```

如需深入分析，系统会调用爬虫 skill 抓取博主主页：
```bash
python /skills/investigation/web-scraper/scrape_page.py \
  --url "https://www.xiaohongshu.com/user/profile/{id}" \
  --output "workspaces/blogger_{id}.md"
```

### 2.4 推荐决策：四维匹配模型

从 10 位中精选 1 位，按 4 个维度评估（见案例报告第四章）：

| 维度 | 评估要点 | 案例（菜菜的减脂日记） |
|------|---------|---------------------|
| **人设匹配** | 博主人设调性是否与品牌一致 | 上班族+减脂双重身份，完美契合目标人群 |
| **内容场景** | 常规内容场景是否适合产品植入 | 便当/早餐场景天然适配酸奶植入 |
| **表达方式** | 文案风格、拍摄手法是否契合品牌调性 | 口语化种草+干货分享，亲和自然 |
| **品牌匹配度** | 合作价值与转化潜力 | 粉丝精准、性价比高、内容可复制性强 |

### 2.5 报告生成

调研结果自动整理为 Markdown 报告，保存到 `analyst/{关键词}_博主调研报告.md`，报告结构：

```
# 小红书博主调研报告
> 调研主题 / 调研日期 / 推荐博主
一、博主筛选清单（10位表格）
二、博主详细信息（逐位展开）
三、推荐博主（四维理由）
四、内容拆解与参考（结构/钩子/镜头/表达 + 创作提示词）
五、合作建议
```

## 三、怎么参考内容（内容拆解方法论）

选定博主后，系统对其近期爆款内容进行**四维拆解**，提炼可复用的创作要素。这是连接"调研"与"脚本创作"的关键桥梁。

### 3.1 内容结构拆解

分析博主的**叙事骨架**——从开场到结尾的内容推进逻辑。

以案例中的菜菜的减脂日记为例，拆解出**"连续剧打卡式"结构**：

```
场景引入（固定格式）→ 食材/营养展示（核心信息）→ 制作/搭配逻辑（实用价值）→ 行动引导（互动闭环）
```

这个结构会直接指导脚本创作的段落划分。

### 3.2 钩子手法拆解

分析博主如何在前 3 秒抓住注意力。常见钩子类型：

| 钩子类型 | 原理 | 案例适配 |
|---------|------|---------|
| 数字承诺钩子 | 用数字制造冲击力 | "上班带饭第30天" |
| 场景共鸣钩子 | 直击目标人群痛点 | "打工人减脂午餐怎么吃" |
| 价值暗示钩子 | 承诺提供实用干货 | 标题末尾"附做法" |
| 反差/悬念钩子 | 制造惊喜感 | "10元搞定高蛋白午餐" |

### 3.3 镜头语言拆解

分析博主的**视觉表达习惯**——景别偏好、转场手法、拍摄角度。

案例拆解出的镜头要素：
- 俯拍便当全景（固定角度，展示完整搭配）
- 食材特写镜头（近景突出质感）
- 制作过程快剪（体现"快手"属性）

这些要素会转化为脚本中的**分镜设计**。

### 3.4 表达方式拆解

分析博主的**语言风格**——口语化程度、专业术语使用、情绪表达。

案例拆解：
- 口语化种草："今天的搭配""明天想吃什么"
- 数据化表达：标注热量、蛋白质含量
- 闺蜜式安利：使用"姐妹们""打工人"等称呼

### 3.5 生成创作提示词

四维拆解的最终产出是一段**可直接用于脚本生成的提示词**（见案例报告第五章末尾），包含：
- 品牌信息（产品/卖点/人群/场景）
- 内容结构参考
- 钩子手法参考
- 镜头语言参考
- 表达方式参考
- 注意事项（合规要求）

这段提示词作为 investigation → script_generate 的**交接物**，确保脚本创作严格遵循博主风格。

## 四、怎么设计脚本和分镜（script_generate 子代理）

### 4.1 脚本生成五步流程

script_generate 子代理接收到 brief + 博主信息 + 内容拆解后，按 5 步生成脚本：

```
步骤1：Brief 拆解 → 步骤2：风格模仿 → 步骤3：脚本生成 → 步骤4：风险质检 → 步骤5：飞书写入
```

### 4.2 Brief 拆解

从品牌 brief 中提取 4 类关键要素：

| 要素 | 内容 | 作用 |
|------|------|------|
| 品牌信息 | 品牌名、行业、调性 | 定调 |
| 产品卖点 | 核心功能点（1-3个）、差异化优势 | 确定植入内容 |
| 目标人群 | 年龄、性别、消费力、痛点 | 确定表达方式 |
| 投放要求 | 植入方式、禁忌词、合规要求 | 约束脚本边界 |

### 4.3 风格模仿

基于 investigation 的内容拆解，提炼 4 个风格要素：

- **叙事结构**：博主的开场→推进→收束方式
- **语言风格**：口语化程度、术语频率、情绪表达
- **镜头习惯**：景别偏好、转场手法、节奏
- **产品露出方式**：博主过往如何自然植入产品

### 4.4 脚本生成：四大模块

生成的完整脚本包含 4 个模块：

#### 模块一：标题与基础信息
```
脚本标题：{小红书爆款标题，数字+emoji+痛点，20字以内}
预计时长：60秒-5分钟
内容类型：好物分享/教程/日常vlog/测评
```

#### 模块二：口播文案（时间线分段）
按时间线分段撰写，每段标注：
- **时间节点**（如 0-5s、5-15s）
- **口播内容**（完整文案，符合博主语言风格，可直接念稿）
- **语气/情绪提示**（如：兴奋、日常、强调）

示例：
```
0-5s | 开场钩子
"姐妹们！上班带饭第28天，今天这个早餐我吃了两周都不腻！"
（语气：兴奋、日常）

5-20s | 产品自然露出
"就是这杯轻醒希腊酸奶，0蔗糖但蛋白质拉满，饱腹感超强..."
（语气：种草、强调）
```

#### 模块三：产品植入点
明确标注产品出现的细节：

| 植入时机 | 植入方式 | 植入话术 | 植入理由 |
|---------|---------|---------|---------|
| 第15秒 | 场景使用 | "今天便当配了这杯0蔗糖酸奶" | 作为便当搭配自然露出，不突兀 |

**植入原则：** 产品作为"日常好物"而非"推广产品"出现，融入博主的常规内容场景。

#### 模块四：分镜设计（逐镜头）

| 镜头 | 时间 | 画面描述 | 口播/字幕 | 景别 | 备注 |
|------|------|---------|-----------|------|------|
| 1 | 0-3s | 俯拍便当全景，酸奶摆在便当盒旁 | "上班带饭第28天" | 全景 | 固定角度 |
| 2 | 3-8s | 酸奶开盖特写，展示浓稠质地 | "0蔗糖高蛋白希腊酸奶" | 特写 | 突出质感 |
| 3 | 8-15s | 酸奶搭配水果/燕麦的制作过程 | "饱腹感直接拉满" | 中景 | 快剪节奏 |
| ... | ... | ... | ... | ... | ... |

**分镜设计原则：**
- 可执行，不需要特殊设备/场景
- 镜头节奏与博主过往内容一致
- 产品露出自然，不喧宾夺主

### 4.5 风险质检：三类检查清单

脚本生成后，必须通过 3 类检查，不通过则修改重检：

#### 合规风险检查
- [ ] 无绝对化用语（"最好""第一""绝对""100%"等违反广告法）
- [ ] 无虚假承诺或夸大宣传
- [ ] 无医疗/健康领域违规功效宣称
- [ ] 产品功效描述有依据支撑
- [ ] 无竞品贬低

#### 内容质量检查
- [ ] 符合博主个人风格，不像硬广
- [ ] 产品植入自然，不突兀
- [ ] 口播文案口语化，可直接念稿录制
- [ ] 分镜可执行，无需特殊设备
- [ ] 开头 3 秒有钩子
- [ ] 时长控制合理

#### 品牌匹配检查
- [ ] 内容调性与品牌一致
- [ ] 卖点传达完整且准确
- [ ] 目标人群能产生共鸣

### 4.6 飞书文档写入

质检通过后，将脚本写入飞书文档，使用 3 个核心飞书工具：

```
create_feishu_document     → 创建文档（标题=脚本标题）
batch_create_feishu_blocks → 批量写入内容块（标题/文本/列表）
create_feishu_table        → 创建分镜表格
```

文档结构：
```
# {脚本标题}
## 基础信息（文本块）
## 口播文案（标题块+文本块）
## 产品植入点（标题块+文本块）
## 分镜设计（标题块+表格块）
## 合规风险提醒（标题块+列表块）
```

## 五、关键设计原则

### 5.1 自然种草，拒绝硬广

贯穿全流程的核心原则。从博主筛选（偏好日常分享型博主）、到内容拆解（提炼自然植入方式）、到脚本生成（产品作为"搭配"而非"主角"），每一步都在抑制硬广感。

### 5.2 风格一致性

脚本必须"像这个博主自己写的"。通过四维内容拆解 + 风格模仿步骤，确保脚本的语言风格、叙事结构、镜头习惯都与博主过往内容一致。

### 5.3 可执行性

脚本必须"博主拿过来就能拍"。分镜设计只使用常规设备可实现的镜头，口播文案可直接念稿，不要求特殊场景或道具。

### 5.4 合规先行

风险质检作为独立步骤，脚本生成后必须通过合规检查才能输出。涵盖广告法、功效宣称、竞品贬低等多个维度。

### 5.5 可复用性

通过 Skill 沉淀（[generate_content SKILL.md](/xiaohongshu_script_assistant/src/workspaces/skills/script_generate/generate_content/SKILL.md)），整套方法论可复用于不同品牌、不同品类的脚本创作，只需更换 brief 输入。
