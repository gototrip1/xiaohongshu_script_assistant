# 小红书短视频脚本生成技能 (Generate Content)

基于品牌 brief 和博主调研结果，生成符合博主风格的、可直接录制的小红书短视频脚本，并写入飞书文档。

## 适用场景

- 已完成博主调研（investigation 子代理产出），需要为选中的博主创作短视频脚本
- 品牌方提供 brief，需要快速产出可执行的短视频内容方案
- 需要将脚本与分镜同步写入飞书文档，方便团队协作

## 输入材料

调用此 Skill 前，需要准备以下输入：

| 输入项 | 必填 | 说明 |
|--------|------|------|
| `brief` | 是 | 品牌需求/产品简介，包含品牌调性、核心卖点、目标人群 |
| `blogger_profile` | 是 | 选定博主的信息（昵称、内容方向、粉丝画像、风格特征） |
| `blogger_reference` | 是 | 博主近期 2-3 条代表内容的拆解（内容结构、钩子、镜头、表达方式） |
| `content_prompt` | 否 | investigation 子代理生成的「内容创作提示词」，作为风格参考 |
| `feishu_folder_token` | 否 | 飞书文件夹 token，用于指定文档保存位置，不填则保存到根目录 |

## 执行步骤

### 步骤 1：Brief 拆解

从 brief 中提取关键要素：

- **品牌信息**：品牌名、行业、调性（高端/平价/专业/亲和）
- **产品卖点**：核心功能点（1-3个）、差异化优势、使用场景
- **目标人群**：年龄段、性别、消费力、痛点
- **投放要求**：软植入/深度种草/测评对比、禁忌词、合规要求

### 步骤 2：内容风格模仿

基于博主调研结果，提炼风格要素：

- **叙事结构**：博主常用的开场方式 → 内容推进逻辑 → 结尾收束方式
- **语言风格**：口语化程度、专业术语使用频率、情绪表达方式
- **镜头习惯**：景别偏好、转场手法、节奏快慢
- **产品露出方式**：博主过往如何自然植入产品（场景融入/使用演示/对比展示）

### 步骤 3：脚本生成

产出完整的短视频脚本，包含以下模块：

#### 3.1 基础信息

```
脚本标题：{吸引眼球的小红书标题}
预计时长：{60-90秒 / 3-5分钟}
内容类型：{好物分享 / 教程 / 日常vlog / 测评}
```

#### 3.2 口播文案

按时间线分段撰写口播文案，每段标注：
- 时间节点（如 0-5s、5-15s）
- 口播内容（符合博主语言风格的完整文案）
- 语气/情绪提示（如：兴奋、日常、强调）

#### 3.3 产品植入点

明确标注产品在脚本中出现的位置和方式：
- 植入时机（第几秒/第几个段落）
- 植入方式（场景使用 / 口播提及 / 画面展示）
- 植入话术（原话）
- 植入理由（为什么在这个位置植入是自然的）

#### 3.4 分镜设计

为脚本提供逐镜头分镜：

| 镜头 | 时间 | 画面描述 | 口播/字幕 | 景别 | 备注 |
|------|------|---------|-----------|------|------|
| 1 | 0-3s | {画面描述} | {口播/字幕内容} | 特写/中景/全景 | {拍摄备注} |
| 2 | 3-8s | ... | ... | ... | ... |
| ... | ... | ... | ... | ... | ... |

### 步骤 4：风险质检

对生成的脚本进行合规与质量检查：

#### 4.1 合规风险检查

- [ ] 无绝对化用语（"最好""第一""绝对"等违反广告法）
- [ ] 无虚假承诺或夸大宣传
- [ ] 无医疗/健康领域的违规功效宣称
- [ ] 产品功效描述有依据支撑
- [ ] 无竞品贬低

#### 4.2 内容质量检查

- [ ] 脚本符合博主个人风格，不像硬广
- [ ] 产品植入自然，不突兀
- [ ] 口播文案口语化，适合直接念稿录制
- [ ] 分镜设计可执行，不需要特殊设备/场景
- [ ] 时长控制合理（60s-5min）
- [ ] 开头 3 秒有钩子，能抓住注意力

#### 4.3 品牌匹配检查

- [ ] 内容调性与品牌一致
- [ ] 卖点传达完整且准确
- [ ] 目标人群能产生共鸣

### 步骤 5：写入飞书文档

将最终脚本写入飞书文档：

1. 使用 `create_feishu_document` 创建文档，标题为脚本标题
2. 使用 `batch_create_feishu_blocks` 写入以下内容：
   - 标题块（脚本标题）
   - 文本块（基础信息）
   - 标题块（口播文案）+ 文本块（分段文案）
   - 标题块（产品植入点）+ 文本块（植入详情）
   - 标题块（分镜设计）+ 表格块（分镜表）
   - 标题块（合规风险提醒）+ 列表块（检查项）
3. 返回飞书文档链接

## 输出格式

最终输出包含两部分：

### 1. 脚本文档内容（Markdown 格式）

```markdown
# {脚本标题}

## 基础信息
- 预计时长：{时长}
- 内容类型：{类型}
- 目标博主：{博主昵称}

## 口播文案

### 0-5s | 开场钩子
{口播内容}
*（语气：{情绪提示}）*

### 5-20s | ...
...

## 产品植入点

| 植入时机 | 植入方式 | 植入话术 | 植入理由 |
|---------|---------|---------|---------|
| 第15秒 | 场景使用 | "{话术}" | {理由} |

## 分镜设计

| 镜头 | 时间 | 画面描述 | 口播/字幕 | 景别 | 备注 |
|------|------|---------|-----------|------|------|
| 1 | 0-3s | ... | ... | ... | ... |

## 合规风险提醒
- ✅ 已检查：无绝对化用语
- ✅ 已检查：无虚假承诺
- ⚠️ 注意：{如有特殊提醒}
```

### 2. 飞书文档写入结果

```
📄 脚本已写入飞书文档
文档标题：{脚本标题}
文档链接：{feishu_url}
```

## 风险检查清单

脚本生成后，必须逐项确认以下检查：

### 合规类
- [ ] 无绝对化用语（最好/第一/绝对/100%等）
- [ ] 无虚假或夸大宣传
- [ ] 功效宣称有依据（尤其化妆品/食品/健康类）
- [ ] 无贬低竞品内容
- [ ] 不涉及医疗诊断/治疗承诺

### 内容类
- [ ] 符合博主风格，非硬广感
- [ ] 产品植入自然不突兀
- [ ] 口播文案可直接录制
- [ ] 分镜可执行、无需特殊设备
- [ ] 开头3秒有吸引力

### 品牌类
- [ ] 调性匹配品牌
- [ ] 卖点传达准确
- [ ] 目标人群可产生共鸣

### 技术类
- [ ] 飞书文档创建成功
- [ ] 分镜表格格式正确
- [ ] 文档链接可访问
