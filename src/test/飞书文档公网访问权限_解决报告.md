## 二、实现方案

### 2.1 核心 API：PATCH 设置公网权限

飞书开放平台提供 `PATCH /open-apis/drive/v1/permissions/{token}/public` 接口更新文档的公共权限设置。

**关键参数说明：**

| 参数 | 设置值 | 含义 |
|------|--------|------|
| `external_access` | `true` | 允许分享到租户（组织）外 |
| `link_share_entity` | `anyone_readable` | **获得链接的任何人可阅读**（公网可访问的关键） |
| `share_entity` | `anyone` | 所有可阅读/编辑的用户可添加协作者 |
| `security_entity` | `anyone_can_view` | 所有可访问用户可创建副本/打印/导出 |
| `comment_entity` | `anyone_can_view` | 所有可访问用户可评论 |
| `invite_external` | `true` | 允许邀请外部人 |

**`link_share_entity` 可选值：**
- `tenant_readable`：组织内获得链接的人可阅读（默认，非公网）
- `tenant_editable`：组织内获得链接的人可编辑
- `anyone_readable`：**获得链接的任何人可阅读**（公网可访问，需 `external_access=true`）
- `anyone_editable`：获得链接的任何人可编辑
- `closed`：关闭链接分享

> 设置 `link_share_entity=anyone_readable` + `external_access=true` 是实现"公网任何人可通过链接访问"的核心。

### 2.2 文档类型（type 参数）

飞书 v1 接口的 `type` 参数可选值：`doc`、`sheet`、`file`、`wiki`、`bitable`。

脚本采用**自动探测策略**：依次尝试 `doc` → `docx` → `sheet` → `wiki`，直到成功（code=0）。遇到 `1063001`（参数不匹配）自动尝试下一个类型，遇到 `1063002`（权限不足）则停止。

### 2.3 公网链接获取

通过 `GET /open-apis/docx/v1/documents/{document_id}` 获取文档信息，响应中的 `document.url` 即为文档的访问链接。设置 `anyone_readable` 后，该链接即为公网可访问链接。

### 2.4 链接可访问性验证

用 `requests.get(doc_url)` 访问链接，判断逻辑：
- HTTP 200 + 页面不含"登录"关键词 → ✅ 公网可访问
- HTTP 200 + 页面含"登录"关键词 → ⚠️ 跳转到登录页，权限未生效
- HTTP 301/302 → ⚠️ 重定向，可能需要登录
- 其他状态码 → ❌ 不可访问

## 三、期间遇到的报错与解决

### 报错 1：ModuleNotFoundError: No module named 'agent'

**现象：**
```
File "test_feishu.py", line 19, in <module>
    from agent.models.env_utils import APP_ID, APP_SECRET, YOUR_OPEN_ID, DOC_TOKEN
ModuleNotFoundError: No module named 'agent'
```

**原因：** 脚本在 `src/test/` 目录下，直接运行时 Python 找不到 `src/` 下的 `agent` 包。

**解决：** 在脚本头部将 `src/` 目录加入 `sys.path`：
```python
SRC_DIR = Path(__file__).resolve().parent.parent
if str(SRC_DIR) not in sys.path:
    sys.path.insert(0, str(SRC_DIR))
```

**状态：** ✅ 已解决

---

### 报错 2：UnicodeEncodeError: 'gbk' codec can't encode character

**现象：**
```
print(f"[1/5] ❌ 获取 token 失败: {e}")
UnicodeEncodeError: 'gbk' codec can't encode character '\u274c' in position 6: illegal multibyte sequence
```

**原因：** Windows PowerShell 默认使用 GBK 编码，无法输出 emoji 字符（❌ \u274c）。

**解决：** 在脚本头部重配置 stdout/stderr 编码为 UTF-8：
```python
if hasattr(sys.stdout, "reconfigure"):
    sys.stdout.reconfigure(encoding="utf-8", errors="replace")
if hasattr(sys.stderr, "reconfigure"):
    sys.stderr.reconfigure(encoding="utf-8", errors="replace")
```

运行时也需设置环境变量：`$env:PYTHONIOENCODING="utf-8"`

**状态：** ✅ 已解决

---

### 报错 3：DNS 解析失败（当前环境网络限制）

**现象：**
```
NameResolutionError: HTTPSConnection(host='open.feishu.cn', port=443): 
Failed to resolve 'open.feishu.cn' ([Errno 11003] getaddrinfo failed)
requests.exceptions.ConnectionError: HTTPSConnectionPool(host='open.feishu.cn', port=443): 
Max retries exceeded with url: /open-apis/auth/v3/tenant_access_token/internal
```

**原因：** 当前 Trae AI 沙箱环境无外网访问权限，DNS 无法解析 `open.feishu.cn`。这是环境限制，非代码问题。

**解决：** 代码已添加异常捕获，在网络错误时优雅输出错误信息而非崩溃。在有网络的本地环境运行即可正常调用飞书 API。

**状态：** ⚠️ 环境限制，需在本地有网络环境运行
